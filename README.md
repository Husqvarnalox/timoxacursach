# Портал «Корочки.есть»

Информационная система для записи на онлайн-курсы дополнительного профессионального образования.
Реализован весь функционал из билета демоэкзамена: регистрация, авторизация, подача заявок,
отзывы, панель администратора.

## 1. Стек технологий и обоснование

| Слой       | Технология                          | Почему именно она                                                                                                          |
|------------|-------------------------------------|----------------------------------------------------------------------------------------------------------------------------|
| Frontend   | HTML + CSS + JavaScript (ванильный) | Требование задания — «максимально просто». Никаких React/Vue/сборщиков.                                                    |
| Backend    | Node.js + Express                   | Браузер не может ходить напрямую в БД — нужен сервер-посредник. Express даёт минимальный роутинг HTTP. Это **не ORM**.     |
| Драйвер БД | `better-sqlite3`                    | Это **не ORM**, а драйвер SQLite. SQL пишется руками: `db.prepare('SELECT ... WHERE id = ?').get(id)`. Синхронный, простой.|
| СУБД       | SQLite                              | Файловая БД, не требует отдельного сервера. Для демо — идеально: вся БД лежит в одном файле `korochki.db`.                 |

### Почему именно Node.js?
- Один язык на фронте и на бэке — JS, не нужно переключаться.
- Минимум зависимостей: всего два пакета (`express`, `better-sqlite3`).
- Запуск одной командой: `npm start`. Под Windows работает идентично macOS/Linux.
- Альтернативы (PHP, Python/Flask) потребовали бы установки дополнительных интерпретаторов.

### Почему SQLite, а не PostgreSQL/MySQL
- **Ноль установки.** Не нужно ставить серверную СУБД, заводить пользователей, выдавать права.
- **Один файл.** Вся БД — это `korochki.db` в папке проекта. Скопировал — переехала на другую машину.
- **Подходит под задачу.** Один пользователь работает за раз, нет высокой нагрузки — SQLite справится с запасом.
- **Та же логика.** Используем те же SQL-запросы: `SELECT`, `INSERT`, `JOIN`, `CHECK`, `FOREIGN KEY`. Если завтра захочется перенести на Postgres — большая часть запросов не изменится.
- **Минусы:** в SQLite нет настоящих параллельных писателей и нет полноценной сетевой работы из коробки. Для демо-проекта ДПО это не нужно.

### Почему без Docker
- SQLite не требует сервера — а Docker нужен в первую очередь чтобы поднимать серверные сервисы. Делать контейнер ради одного `node` процесса — overkill.
- Меньше зависимостей у того, кто будет запускать: достаточно установить Node.js, не нужен Docker Desktop (~500 МБ).
- Запуск становится 2-шаговым: `npm install && npm start`.

## 3. Запуск проекта

### Что нужно установить
**Node.js LTS** (версия 18, 20 или 22) — https://nodejs.org/

- **Windows:** скачать `.msi` установщик, проставить галочки по умолчанию.
- **macOS:** `brew install node` или скачать `.pkg` с сайта.

Проверка: `node -v` должен показать `v18.x` / `v20.x` / `v22.x`.

### Запуск (Windows / macOS / Linux — команды идентичные)

Открыть терминал в папке проекта (на Windows — PowerShell или cmd) и выполнить:

```
npm install
npm start
```

Открыть в браузере: **http://localhost:3000**

### Что делает каждая команда
- `npm install` — скачивает зависимости (`express`, `better-sqlite3`) в папку `node_modules/`. Для `better-sqlite3` ставится прекомпилированный бинарник под вашу ОС — компиляция не нужна.
- `npm start` — запускает сервер на порту 3000. При первом запуске автоматически:
  1. Создаётся файл `korochki.db`.
  2. Выполняется `db/init.sql` — создаются таблицы `users`, `applications`, `reviews` и вставляется демо-пользователь.

### Остановка
В терминале с `npm start` нажать **Ctrl+C**.

### Сброс БД (начать с чистого листа)
Просто удалить файл `korochki.db` и перезапустить `npm start` — он создастся заново.

Windows PowerShell:
```powershell
Remove-Item korochki.db, korochki.db-shm, korochki.db-wal -ErrorAction SilentlyContinue
npm start
```

macOS / Linux:
```bash
rm -f korochki.db korochki.db-shm korochki.db-wal
npm start
```

## 4. Возможные проблемы при установке

### `npm install` падает с ошибкой компиляции better-sqlite3

Скорее всего, используется не-LTS версия Node — для неё нет прекомпиленного бинарника, npm пытается собрать из исходников.

**Решение 1 (рекомендую):** поставить Node LTS (18 / 20 / 22) — для них есть готовые бинарники, компиляция не понадобится.

**Решение 2 (Windows):** установить инструменты сборки. В PowerShell **от имени администратора**:
```powershell
npm install --global windows-build-tools
```
или поставить «Visual Studio Build Tools» вручную с workload’ом «Desktop development with C++».

**Решение 3 (macOS):** установить Xcode Command Line Tools:
```bash
xcode-select --install
```

### Порт 3000 уже занят
Запустить с другим портом:
- Windows PowerShell: `$env:PORT=3001; npm start`
- macOS / Linux: `PORT=3001 npm start`

## 6. Работа с базой данных

### 6.1. Схема (3 таблицы)

```
users                       applications                  reviews
------------                ------------                  -------
id          PK              id              PK            id              PK
login       UNIQUE          user_id         FK→users.id   application_id  FK→applications.id UNIQUE
password                    course_name                   user_id         FK→users.id
full_name                   start_date                    rating          1..5
phone                       payment_method  cash|phone_transfer comment
email                       status          Новая|Идет обучение|Обучение завершено
created_at                  created_at                    created_at
```

Полная схема — в `db/init.sql`.

### 6.2. Подключение к БД для просмотра данных

Поскольку SQLite — это файл, для просмотра нужен любой клиент:

#### Вариант A. Графический клиент (рекомендую)
**DB Browser for SQLite** — бесплатный, простой, кроссплатформенный.
- Скачать: https://sqlitebrowser.org/
- Открыть файл `korochki.db` через File → Open Database.
- Вкладка «Browse Data» — смотреть таблицы.
- Вкладка «Execute SQL» — писать запросы.

#### Вариант B. Командная строка `sqlite3`
- **Windows:** скачать `sqlite-tools-win-x64-*.zip` с https://www.sqlite.org/download.html, распаковать, положить `sqlite3.exe` в папку проекта (или в PATH).
- **macOS:** `sqlite3` уже установлен.

Подключение:
```
sqlite3 korochki.db
```

После этого откроется консоль `sqlite>`.

### 6.3. Полезные SQL-запросы

В sqlite3 CLI:
```sql
-- список таблиц
.tables

-- структура таблицы
.schema users
.schema applications
.schema reviews

-- красивый вывод
.mode column
.headers on

-- все пользователи
SELECT id, login, full_name, phone, email FROM users;

-- все заявки с именем пользователя
SELECT a.id, u.full_name, a.course_name, a.start_date, a.status
FROM applications a
JOIN users u ON u.id = a.user_id
ORDER BY a.created_at DESC;

-- заявки с отзывами
SELECT a.id, a.course_name, a.status, r.rating, r.comment
FROM applications a
LEFT JOIN reviews r ON r.application_id = a.id;

-- ручное изменение статуса (то же, что делает админ-панель)
UPDATE applications SET status = 'Идет обучение' WHERE id = 1;

-- удалить пользователя (каскадно удалит его заявки и отзывы)
DELETE FROM users WHERE login = 'ivanov01';

-- выйти
.quit
```

### 6.4. Сброс БД

Самый простой способ — удалить файл и перезапустить сервер (см. раздел [3. Запуск проекта → Сброс БД](#сброс-бд-начать-с-чистого-листа)).

Можно и в самой БД:
```sql
DROP TABLE reviews;
DROP TABLE applications;
DROP TABLE users;
.quit
```
После перезапуска сервера таблицы создадутся заново из `init.sql`.

### 6.5. Изменение схемы

Правим `db/init.sql`. Поскольку используется `CREATE TABLE IF NOT EXISTS`, для применения изменений к **уже созданной** БД нужно:
- либо удалить файл `korochki.db` (потеряются данные) и запустить сервер;
- либо вручную выполнить `ALTER TABLE ...` через sqlite3 CLI / DB Browser.

## 7. API сервера

Все эндпоинты возвращают JSON.

| Метод  | Путь                              | Описание                                                    |
|--------|-----------------------------------|-------------------------------------------------------------|
| POST   | `/api/register`                   | Регистрация пользователя                                    |
| POST   | `/api/login`                      | Вход (для обычного пользователя и для админа)               |
| GET    | `/api/applications?user_id=N`     | Получить заявки конкретного пользователя                    |
| POST   | `/api/applications`               | Создать новую заявку                                        |
| POST   | `/api/reviews`                    | Оставить/обновить отзыв на свою заявку                      |
| GET    | `/api/admin/applications`         | Все заявки (для админ-панели)                               |
| PATCH  | `/api/admin/applications/:id`     | Сменить статус заявки                                       |

> **Замечание про безопасность.** Пароли хранятся в БД в открытом виде (для простоты демо).
> Авторизация на бэкенде упрощённая: `user_id` передаётся клиентом. В реальной системе
> нужно хешировать пароли (например, `bcrypt`) и проверять сессии/JWT на сервере.

## Сценарий проверки функционала

1. Открыть http://localhost:3000 — увидишь форму входа.
2. Жми «Еще не зарегистрированы? Регистрация», заполни форму, проверь валидации
   (попробуй ввести `qwe` в логин — выдаст ошибку про длину).
3. После регистрации войди под новым пользователем.
4. На странице «Мои заявки» жми «Новая заявка» — заполни форму, отправь.
5. Выйди, войди как `Admin` / `KorokNET` — попадёшь в админ-панель.
6. Поменяй статус заявки на «Идет обучение», затем на «Обучение завершено».
7. Выйди, войди обратно под обычным пользователем — увидишь новый статус,
   оставь отзыв (оценка 1–5 + комментарий).
